# BM25 Retrieval

**Module:** `src/marley/retrieval/`
**Implementation:** `bm25.py`
**Library:** `rank-bm25` (Okapi BM25)

The retrieval module follows a strategy pattern with an abstract `Retriever` base class. This enables plugging in different retrieval strategies (BM25, vector, hybrid) with a consistent interface.

---

## Architecture

```
Retriever (abstract)
├── index(corpus) → None
├── retrieve(query, k) → list[RetrievalResult]
└── size → int

BM25Retriever(Retriever)
└── Uses rank_bm25.BM25Okapi internally

VectorRetriever(Retriever)
└── Uses sentence-transformers + ChromaDB internally (see docs/retrieval/vector.md)
```

---

## Retriever Interface

All retrievers implement three operations:

| Method | Description |
|---|---|
| `index(corpus)` | Build the retrieval index from a list of chunk dicts. Each dict must have `chunk_id`, `text`, and `metadata`. |
| `retrieve(query, k)` | Return top-k results sorted by descending relevance score. |
| `size` | Number of indexed documents. |

Results are returned as `RetrievalResult` dataclasses:

```python
@dataclass
class RetrievalResult:
    chunk_id: str
    text: str
    score: float
    metadata: dict[str, Any]
```

---

## BM25 Implementation

### Text Preprocessing

Tokenization uses lowercase whitespace splitting:

```python
def _tokenize(text: str) -> list[str]:
    return text.lower().split()
```

This simple approach:
- Preserves special tokens like `§23`, `CS 699`, `LP`
- Requires no external stopword lists or language models
- Is fully reproducible and deterministic

### Index Building

1. Load chunks from JSON via `load_chunks(path)`
2. Tokenize each chunk text
3. Build BM25Okapi index over tokenized corpus

### Retrieval

1. Tokenize the query
2. Compute BM25 scores for all documents
3. Sort by score descending
4. Return top-k results with score > 0

### Zero-Score Filtering

Results with BM25 score <= 0 are excluded. This occurs when query terms do not appear in a document or have zero IDF weight.

## Usage

```python
from src.marley.retrieval import BM25Retriever, load_chunks

# Load and index
chunks = load_chunks("data/chunks/stpo-chunks.json")
retriever = BM25Retriever()
retriever.index(chunks)

# Retrieve
results = retriever.retrieve("master thesis credits", k=5)
for r in results:
    print(f"{r.chunk_id}: {r.score:.2f}")
```

### Loading Multiple Knowledge Bases

```python
stpo = load_chunks("data/chunks/stpo-chunks.json")
faq_stpo = load_chunks("data/chunks/faq-stpo-chunks.json")

# Combined index
retriever = BM25Retriever()
retriever.index(stpo + faq_stpo)
```

## Configuration

| Parameter | Default | Description |
|---|---|---|
| `k` | 5 | Number of top results to return |
| Tokenizer | lowercase + split | Text preprocessing strategy |

BM25 hyperparameters (`k1`, `b`) use the rank_bm25 defaults (k1=1.5, b=0.75).
